# スケジュール画像作成アプリ

キャストのシフトスケジュールを入力し、PNG画像として書き出すWebアプリです。

## デプロイ

GitHub Pages でホストされています。`main` ブランチに push すると自動デプロイされます。

- ワークフロー: `.github/workflows/deploy.yml`
- ベースパス: `/cast-shift-app/`

## ローカル起動

```bash
npm install
npm run dev
```

http://localhost:3000 で開きます。

## STEP 1｜キャストマスターを確認する

キャスト情報（名前・ランク）は **Google スプレッドシート** で一元管理されています。
キャスト画像は **Google Drive フォルダ** にキャスト名と同じファイル名で保存します。

アプリ起動時にGAS（Google Apps Script）WebアプリがシートとDriveフォルダを読み取り、キャスト名と画像を自動マッチングしてアプリに返します。

- 「キャストマスター」タブ → 「参照スプレッドシートを開く」リンクからシートを編集できます
- 画像の追加・入れ替えはDriveフォルダにファイルを入れるだけ（キャスト名.jpg/png）
- キャストの追加・変更はシート側で行い、アプリをリロードすると反映されます
- ネットワークエラー時は前回のキャッシュが使われます

### データソース

| データ | 管理場所 |
|--------|----------|
| 名前 | スプレッドシート（キャストリストタブ） |
| ランク | スプレッドシート（キャストリストタブ） |
| 画像 | Google Drive フォルダ（キャスト名で自動マッチ） |

### GAS Webアプリ

- `gas/castListApi.gs` にコードがあります
- GET: シート読み取り + Driveフォルダ画像マッチング → JSON返却
- POST: `uploadImage`（Driveにアップロード）、`fetchImages`（fileId→Base64取得）
- アップロード先フォルダは `_data` シートの C2 セルに設定

---

## STEP 2｜ランクを設定する

ランクはスプレッドシートのキャストリストタブで管理します。

| ランク | 表示 |
|--------|------|
| GOLD | 金色バッジ |
| SILVER | 銀色バッジ |
| BRONZE | 銅色バッジ |
| （未設定） | バッジなし |

ランクはスケジュール画像上の表示順・装飾に影響します。

---

## STEP 3｜スケジュールを入力する

「スケジュール編集」タブを使います。

### 入力方法①：XLSXファイルから読み込む

「XLSX読込」ボタンから `.xlsx` / `.xls` ファイルを選択します。
7日分まで一度に読み込めます。

### 入力方法②：時間帯ごとに個別入力

「時間帯ごとのキャスト」セクションで時間帯（11:00〜24:00）をクリックして展開し、
キャスト名をカンマまたは改行区切りで入力して「一括追加」を押します。

- 各時間帯のキャストは個別に削除・クリアできます

---

## STEP 4｜ロゴ画像を設定する（任意）

「スケジュール編集」タブ下部の「ロゴ画像」欄から画像ファイルを設定すると、
タイムライン画像にロゴが表示されます。

---

## STEP 5｜プレビューを確認してPNG書き出し

### プレビューモード

右側パネル上部のタブで切り替えます。

| モード | 説明 |
|--------|------|
| 一覧（タイムライン） | 時間軸上にキャストのアイコンを並べた形式 |
| 表形式 | キャストと時間帯をクロス表で表示 |

タイムラインモードでは **16:9 / 1:1** のアスペクト比を選択できます。

### 日付の切り替え

XLSX読込で複数日分を取り込んだ場合、「◀ ▶」ボタンで日付を切り替えられます。

### PNG書き出し・Driveアップロード

| 操作 | 動作 |
|------|------|
| 「保存 & Drive送信」ボタン | PNG保存 + Google Driveにアップロード（確認トーストあり） |
| 「週一括PNG出力（N日分）」 | 全日付を16:9・1:1の両方でZIPにまとめて保存 |

保存時に確認トーストが表示されます。「キャンセル」を選ぶとDriveアップロードもスキップされます。

---

## 画像ロードの仕組み

起動時に3段階で画像をロードします。

1. **HTTP直接取得**: Google DriveのサムネイルURLでブラウザから直接取得（高速）
2. **GASフォールバック**: HTTP取得に失敗した画像はGAS経由でBase64取得（確実）
3. **バックグラウンドプリフェッチ**: メイン画面表示後、全画像のBase64をGAS経由で事前取得（エクスポート高速化）

Canvas描画時はプリロード済みキャッシュから即座に描画します。

---

## データの保存について

すべてのデータはブラウザの **localStorage** に自動保存されます。
ページを閉じても次回開いたときに復元されます。

| 保存されるデータ |
|----------------|
| キャストマスター（GASからのキャッシュ） |
| ランクリスト（ゴールド・シルバー・ブロンズ） |
| スケジュール（日付ごとの時間帯・キャスト） |
| ロゴ画像 |

> ブラウザのキャッシュをクリアするとデータが消えます。

---

## キャスト名の正規化ルール

スケジュール入力時のキャスト名は自動的に正規化されてマスターと照合されます。
以下の違いは同一人物として扱われます。

- 全角・半角スペースの違い
- 先頭の記号（`_` `-` `・` `.` など）の有無
- Unicode正規化の違い（濁点の合成方式など）

GAS側でも画像ファイル名とキャスト名の照合時にスペースの揺れを吸収します。

## よくある問題

| 症状 | 対処法 |
|------|--------|
| 画像が表示されない | Driveフォルダにキャスト名と同じファイル名の画像があるか確認してください |
| 画像フォルダの画像が読めない | フォルダが「リンクを知っている全員」に共有されているか確認してください |
| キャスト名が一致しない | 名前の表記ゆれを確認し、シート側の名前に統一してください |
| 保存容量エラーが出る | 不要なスケジュールデータを削除してください |
| 起動時にローディングが終わらない | ネットワーク接続を確認し、ページをリロードしてください |
| Driveアップロードが失敗する | GASの権限を確認してください（Drive書き込み権限が必要） |
